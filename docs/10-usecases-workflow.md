# 10. 汎用申請ユースケース

## UC-W001: 申請種別を作成する

1. 管理者が申請種別を作成する
2. 名称、説明、入力項目、添付必須有無を設定する
3. バックオフィスタスクを生成するか設定する
4. 申請可能な対象者を設定する
5. 保存する

### 申請種別例

- 経費精算
- 交通費精算
- 名刺申請
- 備品申請
- グッズ申請
- 住所変更
- 通勤経路変更
- 証明書発行
- アカウント発行
- 一般申請

## UC-W002: 社員が申請する

1. 社員が申請種別を選択する
2. フォームに入力する
3. 添付ファイルを追加する
4. 承認者を任意の社員から選択する
5. 下書き保存または申請する
6. 申請イベントを記録する
7. 承認者へ Teams 通知する

## UC-W003: 承認者が申請を承認する

1. 承認者が申請詳細を開く
2. 申請内容、添付ファイル、履歴を確認する
3. 承認する
4. 最終承認なら申請ステータスを承認済みにする
5. バックオフィスタスクが必要な申請種別ならタスクを作成する
6. 申請者へ通知する

## UC-W004: 承認者が差戻しする

1. 承認者が不備を確認する
2. コメントを入力する
3. 差戻しする
4. 申請者へ通知する
5. 申請者は修正して再申請する

## UC-W005: 申請者が申請を取り消す

1. 申請者が自分の申請を開く
2. 取消可能なステータスであることを確認する
3. 取消理由を入力する
4. 取り消す
5. 承認者へ通知する

## 実装上のポイント

- 申請種別 (`request_types`) はマスタ化し、フォーム項目は `form_schema` (JSON) で動的に定義する。
  新しい申請種別を追加する際にコード変更を必要最小限にする
  (スキル [add-workflow-request-type](../.claude/skills/add-workflow-request-type/SKILL.md) 参照)。
- 承認者は固定ルートではなく、申請ごとに任意の社員を `approver_user_id` として指定する。
- `requires_backoffice_task` が true の申請種別は、最終承認時に
  [11章 バックオフィス処理](./11-usecases-backoffice.md) の UC-B001 でタスクを自動生成する。
- 添付ファイルは [12章](./12-usecases-attachment.md) の仕組みを共通利用する。
