# 12. 添付ファイル管理ユースケース

## UC-F001: ファイルを添付する

1. ユーザーがファイルを選択する
2. サイズ、拡張子、MIMEタイプを検証する
3. ストレージに保存する
4. 申請または勤怠に紐づける
5. アップロードイベントを記録する

## UC-F002: 添付ファイルを閲覧する

1. 権限を確認する
2. 閲覧可能ならファイルを表示またはダウンロードする
3. 閲覧ログを記録する

## 実装上のポイント

- `attachments.owner_type` / `owner_id` のポリモーフィック関連で、申請・勤怠など任意の
  エンティティに添付できるようにする。
- サイズ・拡張子・MIMEタイプの許可リストはマスタ化し、申請種別ごとに上書き可能にする
  (経費精算はPDF/画像のみ等)。
- ダウンロード時は必ず所有者・関係者(申請者/承認者/バックオフィス担当者)であることを
  認可チェックし、閲覧ログを [監査ログ](./15-usecases-admin.md#uc-m003-監査ログを確認する) に残す。
- 実ファイルはXSERVER上のストレージまたは外部ストレージに保存し、`stored_path` はDB直下の
  Webルートから直接アクセスできない場所に置く。
