# 22. 用語集

| 用語 | 意味 |
|---|---|
| Command | 状態変更の意図を表すオブジェクト。CommandHandlerに渡される。 |
| CommandHandler | Commandを受け取り、業務ルールを検証してイベントを`stored_events`に記録する処理。 |
| EventStore | `stored_events`テーブル。全ドメインイベントの正。 |
| Projector | イベントを購読し、Projection Tableを更新する処理。 |
| Projection | 画面表示用に非正規化された読み取り専用データ。イベントから再生成可能。 |
| Aggregate | イベントの単位となる集約(例: 1人の1日の勤怠 = attendance_day)。 |
| 所定労働時間 | 勤務形態ごとに定められた基準の労働時間。 |
| 法定休日 | 労働基準法上の休日(週1日または4週4日)。 |
| 所定休日 | 会社が定める休日(法定休日以外の休み)。 |
| 継続勤務期間 | 入社日からの在籍継続期間。有給付与日数の算定に使う。 |
| 出勤率 | 有給付与要件となる、全労働日に対する出勤日の割合(原則8割以上)。 |
| 有給消化 | 有給休暇を使用すること。有効期限が近い付与分から優先消化する。 |
| バックオフィスタスク | 承認済み申請に対して発生する事務処理(発注・支払・発送など)。承認とは別ステータスで管理。 |
| 締め | 対象月の勤怠を確定し、日次勤怠をロックする処理。締め後は修正申請でのみ変更可能。 |
| 修正申請 | 締め後の勤怠を修正するための申請ワークフロー(通常の日次編集とは別)。 |
